## Structure

### Feature-based architecture.

#### app/

- root app components like App.tsx, router.
- providers like AuthProvider

#### features/

- business logic split by domain
- subfolders can be eg. consultant/ or sales/
- each feature has their components, pages and styling.

#### shared/

- components used in multiple features.

#### styles/

- app-wide styling, for example the MUI theme.

### layouts/

- layouts for the app and different pages.

## Using API types in frontend

The frontend uses **generated OpenAPI types** for all API calls.
These types are generated by the backend and are located in
shared/generated/api-types/openapi.d.ts

- Do not edit this manually

Usage example:
import type { paths } from "@api-types/openapi";

type LoginRequest =
paths["/auth/login"]["post"]["requestBody"]["content"]["application/json"];

type LoginResponse =
paths["/auth/login"]["post"]["responses"]["200"]["content"]["application/json"];

example call:
async function login(data: LoginRequest): Promise<LoginResponse> {
const res = await fetch("/api/auth/login", {
method: "POST",
headers: { "Content-Type": "application/json" },
body: JSON.stringify(data),
});

return res.json();
}

This is just for ensuring that the calls are in right format, the validation will be dealt in backend however. So whether to use types is up to you.
